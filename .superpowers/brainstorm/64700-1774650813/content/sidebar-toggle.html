A — Tab Bar at top of sidebar

Two persistent tabs — "Files" and "Search" — always visible at the top of the left pane. One tap to switch. Active tab underlined in Nomad Blue.

Pros

  • Always discoverable
  • Clear active state
  • VS Code Activity Bar feel

Cons

  • Takes ~28px from the sidebar

B — Search icon in sidebar header

A 🔍 icon sits in the file explorer header row. Tapping it swaps the panel to search mode; a ✕ or "back" icon returns to the file tree.

Pros

  • No extra chrome — sidebar header stays compact
  • Clean, minimal UI

Cons

  • Less obvious than a tab bar
  • Requires back-navigation to return to files

C — Dedicated FAB (floating button)

A 🔍 FAB joins the existing ⌘ and >_ buttons. Tapping it opens search in the left sidebar. Same pattern as the terminal toggle.

Pros

  • Consistent with existing FAB pattern
  • Always reachable, no sidebar needed

Cons

  • Three FABs starts feeling crowded
  • Less obvious that it lives in the sidebar
