A — Left Sidebar Panel

Search panel replaces the file explorer in the left pane. Results stay visible while you navigate. Familiar from VS Code's sidebar search.

Pros

  • Results stay visible alongside editor
  • VS Code muscle memory
  • Ideal for tablet landscape

Cons

  • Hides file tree while searching
  • Needs a toggle to switch back

B — Floating Modal

Full-width modal overlay, centred on screen. Dismissed with Escape or tapping outside. Results shown inline. Works the same on phone and tablet.

Pros

  • Consistent with CommandPalette pattern already in app
  • Works well at all screen sizes
  • Fast to open/close

Cons

  • Covers editor while browsing results
  • Must navigate away to see file context

C — Bottom Sheet

Slides up from the bottom, like the terminal. Search bar at top of sheet, results scrollable below. More mobile-native feel.

Pros

  • Native mobile UX pattern
  • Editor still partially visible above
  • Consistent with terminal panel

Cons

  • Less screen real estate for results
  • Not as familiar for developers
